Authentication (passwordless · email OTP)

Sign up with email + password → enter the 6-digit code → land on a session via auto-sign-in. Returning users sign in by entering their email and the OTP — no password prompt at all (Cognito's USER_AUTH flow with EMAIL_OTP as the preferred first factor).

Role-Gated Endpoints

The backend wires auth.requireRole(context, '<group>') in front of these endpoints. Freshly signed-up users belong to no groups, so both calls return 403 NotAuthorizedException out of the box — that's the wire-up working correctly.

To see the success path, add your user to a group out-of-band — in production that means the AWS Console (Cognito → User pools → Users → Add to group) or the CLI: 

aws cognito-idp admin-add-user-to-group \
  --user-pool-id <pool-id> \
  --username <username> \
  --group-name editors

Sign out and back in afterwards so the new ID token reflects the membership. (Group management from application code is coming in a dedicated Building Block.)
